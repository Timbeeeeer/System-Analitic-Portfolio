Название БД: Analytical_data
Поля:
    1)Id - PK (Айди данных в базе)
    2)order_id - int (Номер заказа с FunPay)
    3)price_order - int (Сумма заказа)
    4)date_create - date (Дата когда создан)
    5)date_completed - date (Дата когда выполнен)
    6)status_id - FK (Для скрепления баз данных)
    7)customers_id - FK (Для скрепления баз данных)
    8)TypeOrder_id - FK (Для скрепления баз данных)

Название БД: status_order
Поля:
     1)Id (Айди данных в базе)
     2)status_id - PK (Для скрепления бд)
     3)status - text (Наименование статусов)
        Типы статусов:
        1) completed (Выполнен)
        2) cancelled (Отменён)
        3) Awaiting (Ожидает)

Название БД: customers
Поля:
     1)customers_id - PK (Для скрепления баз данных)
     2)name - text (Имя пользователя)
     
Название БД: order_type
Поля:
    1) TypeOrder_id - PK (Для скрепления баз данных)
    2)type - text (Наименование типов заказов)
        Типы заказов:
        1)Boost (Буст)
        2)Currency (Игровая Валюта)
    

Тип баз данных один ко многим
Таблица Analytical_data связывается с таблицей status_order по полю *status_id*
Таблица Analytical_data связывается с таблицей customers по полю *customers_id*
Таблица Analytical_data связывается с таблицей order_type по полю *TypeOrder_id*
