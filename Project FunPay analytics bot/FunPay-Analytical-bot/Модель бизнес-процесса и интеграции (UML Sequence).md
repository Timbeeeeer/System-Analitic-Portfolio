Так как официальное API у платформы FunPay отсутствует, интеграция реализована методом веб-скрейпинга (партинг DOM-дерева страницы истории продаж).

Для защиты сессии пользователя от блокировок антифрод-системами (Cloudflare Rate Limits) на уровне архитектуры заложен паттерн кэширования: бот обращается к сайту только при отсутствии актуальных данных в локальной БД.

UML диаграмму можно проверить в папке diagrams

Алгоритм ETL и бизнес-правила маппинга:
Модуль парсинга (запускаемый асинхронно) должен выполнять три последовательных шага: Извлечение, Трансформация и Загрузка данных.

    Извлечение (Extract):
        Скрипт выполняет HTTP GET запрос на страницу `https://funpay.com`, используя токен авторизации (куку `golden_key`) в заголовках запроса.

    Загрузка (Load)
        Данные записываются в таблицу `Analytical_data`. Для предотвращения дублирования финансовых показателей при повторном парсинге одной и той же страницы используется логика `INSERT OR IGNORE` по уникальному полю `order_id`.
